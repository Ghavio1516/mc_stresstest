# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Development Commands

- **Run the performance tester (interactive menu)**:  
  `python minecraft_tester.py`  
  or on Windows: double-click `run_test.bat`

- **Run a specific scenario non-interactively**:  
  Edit `minecraft_tester.py` to call `tester.run_test("<scenario>")` directly, or modify the script to accept command-line arguments.

- **Run a single real moving bot manually (for debugging)**:  
  `node wander_bot.js --host <host> --port <port> --username <name>`  
  Example: `node wander_bot.js --host basic-1.alstore.space --port 25641 --username W_Test01`

- **Install Node.js dependencies (if needed)**:  
  `npm install` (installs mineflayer and mineflayer-pathfinder from package.json)

- **View test results**:  
  Check the `results/` directory for CSV summary files and JSON detailed outputs.

- **Stop all running bots (emergency)**:  
  Press `Ctrl+C` in the terminal running the tester, or manually kill Node processes.

## Project Structure & Architecture

- **minecraft_tester.py** – Main orchestrator. Handles scenario selection, bot spawning (via `SequentialBotManager`), test timing, monitoring, and report generation. Uses `EnhancedPerformanceTester` to load configuration and execute tests.
- **wander_bot.js** – Mineflayer-based bot that connects to the Minecraft server, performs random walk with pathfinding to generate chunk load and world generation. Logs its own stats to `results/bot_stats/W_<id>.json` on disconnect.
- **config.json** – Defines Minecraft server connection details (host, ports, RCON) and predefined test scenarios (quick, medium, heavy, extreme) with player count, duration, and description.
- **results/** – Output folder:  
  - `test_summary_*.csv` – Aggregated metrics per scenario (TPS, chunks/sec, entities/sec, etc.).  
  - `detailed_results_*.json` – Full per‑bot data and timestamps.  
  - `bot_stats/` – Individual bot JSON stats files generated by `wander_bot.js`.

### Key Classes (minecraft_tester.py)

- `SequentialBotManager` – Spawns bots one‑by‑one with a configurable delay between each spawn. Designed for servers with very strict connection throttling. Each bot runs in its own Node process (`wander_bot.js`).
- `EnhancedPerformanceTester` – Reads `config.json`, runs a selected scenario, manages the bot manager, and generates final reports.

### Bot Behavior (wander_bot.js)

- Connects in offline mode (`auth: 'offline'`) with protocol version `1.21.8`.
- Uses `mineflayer-pathfinder` to pick random targets within a radius, walk toward them, and thereby force chunk loading.
- Spawn delay between bots is handled by the orchestrator (default 6 seconds in sequential mode) to avoid login throttling.
- On disconnect, writes a JSON file containing distance walked, unique chunks visited, and timestamps.

### Configuration Notes

- The server must have `online-mode=false` and `enforce-secure-profile=false` in `server.properties`.
- RCON should be enabled (`enable-rcon=true`) for real‑time TPS monitoring; if unavailable, the tester falls back to simulated TPS.
- Adjust `bot_mode` in `config.json` (currently `"optimized"`); the orchestrator always uses sequential spawning regardless of this value (the variable is reserved for future extensions).

### Typical Workflow

1. Ensure the Minecraft server is running with the required properties.
2. Update `config.json` with your server’s host, ports, and RCON password.
3. Install Node dependencies if not already present (`npm install`).
4. Launch the tester via `python minecraft_tester.py` (or `run_test.bat`).
5. Choose a scenario from the menu or modify the code to run a specific test.
6. After the test completes, review the generated CSV/JSON files in `results/`.
